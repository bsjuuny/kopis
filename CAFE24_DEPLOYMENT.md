# Cafe24 호스팅 배포 가이드

## 📋 준비사항

Cafe24 리눅스 호스팅에서는 **PHP 프록시 파일**을 사용하여 CORS 문제를 해결합니다.

## 🚀 배포 방법

### 1단계: 빌드

```bash
npm run build
```

### 2단계: dist 폴더 업로드

FTP 또는 Cafe24 파일 관리자를 사용하여 `dist` 폴더의 **모든 파일**을 서버의 `/kopis/` 디렉토리에 업로드하세요.

**중요**: 다음 파일들을 반드시 포함해야 합니다!

```
서버 경로: /home/계정명/www/kopis/
업로드할 파일:
  ✓ index.html
  ✓ .htaccess (숨김 파일 - 클라이언트 라우팅 처리)
  ✓ proxy.php (PHP 프록시 - CORS 해결)
  ✓ assets/ (폴더)
```

### 3단계: 파일 권한 설정

Cafe24 파일 관리자에서:
- `.htaccess` 파일 권한: 644
- `proxy.php` 파일 권한: 644 또는 755
- `index.html` 파일 권한: 644
- `assets/` 폴더 권한: 755

## 🔧 설정 파일

### .htaccess
클라이언트 사이드 라우팅을 처리하여 404 에러를 방지합니다.

### proxy.php
KOPIS API 요청을 중계하여 CORS 문제를 해결합니다. 이 파일은:
- CORS 헤더를 추가하여 브라우저 제한 우회
- KOPIS API로 요청을 전달
- 응답을 클라이언트에 반환

## ✅ CORS 문제 해결

Cafe24는 PHP를 지원하므로, **자체 PHP 프록시 파일**을 사용합니다:

- **장점**: 안정적이고 빠름, 외부 서비스 의존성 없음
- **단점**: 없음 (Cafe24는 PHP를 기본 지원)

### 작동 방식

```
브라우저 → /kopis/proxy.php → KOPIS API → 응답 → 브라우저
```

`proxy.php`가 중간에서 CORS 헤더를 추가하여 브라우저가 응답을 받을 수 있게 합니다.

## 🌐 접속 URL

배포 후 다음 URL로 접속:
```
https://your-domain.com/kopis/
```

## 🐛 문제 해결

### 404 에러가 계속 발생하는 경우

1. `.htaccess` 파일이 업로드되었는지 확인
2. FTP 클라이언트에서 "숨김 파일 표시" 옵션 활성화
3. Cafe24 파일 관리자에서 직접 `.htaccess` 파일 확인

### API 호출이 실패하는 경우

1. 브라우저 개발자 도구 → Network 탭에서 API 요청 확인
2. CORS 프록시 서비스가 정상 작동하는지 확인
3. API 키가 올바른지 확인 (.env 파일의 VITE_KOPIS_API_KEY)

### 페이지가 깨져 보이는 경우

1. `vite.config.js`의 `base: '/kopis/'` 설정 확인
2. 서버 경로가 `/kopis/`인지 확인
3. 브라우저 캐시 삭제 후 새로고침 (Ctrl+Shift+R)

## 📝 FTP 업로드 팁

### FileZilla 사용 시

1. **숨김 파일 표시**
   - 서버 → 강제로 숨김 파일 표시

2. **.htaccess 파일 전송**
   - 파일 → 전송 설정 → 파일 형식: ASCII

3. **대량 파일 업로드**
   - `dist` 폴더 전체를 드래그하여 `/kopis/` 폴더에 드롭

## 🔄 업데이트 방법

코드 수정 후:

```bash
npm run build
```

그리고 `dist` 폴더의 파일들을 다시 업로드하세요.

## ⚡ 성능 최적화

`.htaccess` 파일에 캐싱 설정이 포함되어 있어:
- 이미지: 1년 캐싱
- CSS/JS: 1개월 캐싱

이로 인해 두 번째 방문부터는 로딩이 매우 빠릅니다.

## 💡 주의사항

1. **CORS 프록시 서비스의 한계**
   - 무료 서비스는 속도가 느릴 수 있습니다
   - 안정성이 보장되지 않습니다
   - 프로덕션 환경에서는 자체 프록시 서버 권장

2. **더 나은 대안**
   - Vercel, Netlify 등의 플랫폼 사용 (무료, 프록시 지원)
   - VPS 서버에서 Express 서버 실행

하지만 Cafe24에서 간단히 테스트하거나 소규모로 운영하기에는 충분합니다!
